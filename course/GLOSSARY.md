# Глоссарий терминов курса

Этот глоссарий содержит определения ключевых терминов, используемых в курсе "AI для разработчиков: от копилота до оркестрации агентов". Термины расположены в алфавитном порядке.

---

## A

**ADR (Architecture Decision Record)** — документ, фиксирующий важное архитектурное решение, его контекст, рассмотренные альтернативы и последствия. См. Модуль 5.

**AGENTS.md** — конфигурационный файл в корне проекта с инструкциями для AI-агента. Содержит tech stack, conventions, правила разработки и запреты. Агент читает этот файл перед началом работы. См. Модули 4 и 5.

**AI-IDE** — интегрированная среда разработки со встроенным AI-агентом (Cursor, Antigravity), которая может автономно работать с кодовой базой — читать файлы, редактировать код, запускать команды. См. Модуль 2.

**Antipatterns** — запрещённые паттерны разработки, которые не должны использоваться в проекте. Фиксируются в constitution для предотвращения повторения ошибок. См. Модуль 5.

## B

**Batch API** — режим отправки задач пакетом с получением результата через 1-24 часа со скидкой 50%. Подходит для несрочных задач. См. Модуль 2.

## C

**Chain-of-thought** — техника промптинга, при которой модель "думает вслух", показывая промежуточные шаги рассуждений. Улучшает качество ответов на сложных задачах. См. Модуль 3.

**Claude Code** — официальный CLI-агент от Anthropic для автономной разработки. Работает с файлами, git, терминалом. См. Модули 2 и 4.

**CLAUDE.md** — альтернативное название AGENTS.md, специфичное для Claude Code. См. Модуль 4.

**Constitution** — набор правил, принципов и ограничений для AI-агента, определяющих что можно и нельзя делать в проекте. Хранится в файлах типа constitution.md или AGENTS.md. См. Модули 4 и 5.

**Context Engineering** — управление тем, какая информация попадает в контекстное окно агента. Включает структурирование артефактов, сжатие контекста, управление памятью. См. Модуль 5.

## D

**Decision Log** — журнал значимых технических решений с контекстом, рассмотренными альтернативами и последствиями. См. Модуль 5.

**DeepSeek** — open-source модели V3 и R1 от китайской компании, предлагающие reasoning по цене на 95% дешевле o1. См. Модуль 2.

## E

**Enji Fleet** — внутренний проект Mad Devs по созданию платформы оркестрации AI-агентов, используемый в курсе как демонстрация best practices. См. Модули 1, 5, 7.

## F

**Fail-until-done** — подход, при котором агент работает в цикле до достижения успеха (все тесты проходят), не останавливаясь при первой ошибке. См. Модуль 7.

**Few-shot промптинг** — техника, при которой в промпте даются несколько примеров желаемого результата, чтобы модель поняла паттерн и применила его. См. Модуль 3.

**Forward reference** — упоминание термина до его определения в тексте, создающее путаницу для читателя. Например, если модуль 3 упоминает «SDD» до его определения в модуле 5.

## H

**Headless-режим** — запуск Claude Code без UI, через командную строку. Используется для автоматизации и работы на удалённых серверах. См. Модуль 7.

**HITL (Human-in-the-loop)** — режим работы, при котором агент запрашивает подтверждение на каждое действие. Используется для критичного кода. См. Модуль 4.

**Hot-swap** — автоматическая замена зациклившегося или неэффективного агента другим агентом с той же ролью. См. Модуль 7.

## M

**MCP (Model Context Protocol)** — стандарт подключения внешних инструментов и сервисов к AI-агенту (Jira, БД, браузер, Figma). См. Модуль 6.

**Memory Bank** — директория с тематическими файлами (auth.md, database.md), которые агент читает по необходимости вместо загрузки всей constitution. См. Модуль 5.

## P

**Plan** — документ с подзадачами, созданный агентом-планировщиком для выполнения фичи. Содержит последовательность шагов, файлы для изменения, критерии приёмки. См. Модули 4 и 5.

**Prompt Caching** — кэширование длинного контекста моделью для экономии 90% на повторных запросах. См. Модуль 2.

## R

**RAG (Retrieval-Augmented Generation)** — техника дополнения контекста агента данными из внешних источников (поиск по базе знаний, документации). См. Модуль 1.

**Ralph Loop** — паттерн автономной работы агента: Do (выполнить) → Check (проверить тестами/линтером) → Fix (исправить ошибки) → повторять до успеха. Также известен как fail-until-done. См. Модуль 7.

**Reflect-mode** — специализированный агент, который читает traces от других агентов и консолидирует знания в constitution. Не пишет код, только анализирует и обновляет документацию. См. Модуль 5.

**Runbook** — формализованный план работы для нескольких агентов с ролями, порядком выполнения и критериями приёмки. См. Модуль 7.

## S

**SDD (Spec-Driven Development)** — методология разработки с AI-агентами, при которой сначала пишется детальная спецификация, а затем агент реализует её автономно. См. Модуль 5.

**Skills** — готовые переиспользуемые промпты для повторяющихся задач (code review, тестирование, SDD workflow). Вызываются slash-командами. См. Модуль 5.

**Steering** — управление направлением работы агента через серию checkpoint'ов: research → plan → implement → verify. На каждом этапе возможна корректировка. См. Модуль 4.

## T

**TDD (Test-Driven Development)** — методология разработки, при которой сначала пишутся тесты, затем код для их прохождения. Особенно эффективна с AI-агентами. См. Модуль 4.

**Trace** — документ, который агент создаёт после выполнения задачи. Фиксирует принятые решения, проблемы и их решения, предложения для constitution. См. Модуль 5.

## W

**Worktree (git worktree)** — механизм git для создания нескольких рабочих директорий из одного репозитория. Используется для изоляции контекстов параллельных агентов. См. Модуль 7.

## Y

**YOLO mode** — режим работы агента без подтверждений (You Only Live Once), когда агент работает полностью автономно. Противоположность HITL. См. Модули 2 и 8.

---

## Как пользоваться глоссарием

- **При первом чтении курса:** используйте глоссарий для быстрого поиска незнакомых терминов
- **При выполнении упражнений:** обращайтесь к глоссарию, если забыли значение термина
- **При ревью кода:** проверьте, что используете термины правильно согласно глоссарию

Если термин встретился в тексте, но отсутствует в глоссарии — это ошибка. Сообщите об этом преподавателю.
