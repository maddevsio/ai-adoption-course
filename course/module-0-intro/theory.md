# Введение в курс: От чата до оркестрации агентов

## О чём этот курс

Курс про интеграцию ИИ в рабочий процесс разработчика.

Цель - научиться:
- Выбирать правильный ИИ-инструмент под конкретную задачу
- Делегировать агентам целые фичи, а не только фрагменты кода
- Расширять возможности агентов через MCP-серверы
- Писать спецификации, по которым агент работает автономно
- Координировать несколько агентов параллельно для решения сложных задач
- Понимать риски и границы ответственности на каждом уровне автономности

К концу курса у вас будет полный рабочий сетап:
- AI-IDE или CLI-агент
- MCP-серверы
- Настроенные правила работы агента
- Понимание паттернов оркестрации

Курс ведёт вас с уровня 1 (использую ChatGPT для вопросов) до уровня 5 (координирую несколько агентов).

## Как устроен курс

Курс состоит из 8 модулей. Каждый модуль — пара "теория + практика". Теория занимает ~3 часа, практика ~5 часов.

### Как работать с практикой

**Важно:** Практические задания рассчитаны на ваш реальный проект. Если у вас нет подходящего проекта, используйте пет-проект МД или форк опенсорсного проекта

Каждое задание — действие на вашей машине:
- Установить инструмент → проверить, что работает
- Поставить задачу агенту → получить результат → проверить качество
- Настроить MCP → убедиться, что агент может ходить во внешние системы
- Запустить 2 агентов параллельно → объединить результаты

### Как оценивать прогресс

**В начале курса (модуль 1):** Пройдите самодиагностику — определите свой текущий уровень зрелости (1-5).

**В конце каждой части (модули 3, 5, 7, 8):** Чек-лист навыков. Проверьте, что освоили ключевые практики перед переходом дальше.

**В конце курса (модуль 8):** Повторная самодиагностика — сравните результат с началом. Финальный тест на понимание рисков и ответственности.

**Критерий успеха:** Вы делегируете агенту реальную задачу из проекта (например, "добавь CRUD для модели Product"). Агент выдаёт рабочий результат с минимальными правками. Если это работает — курс пройден.

## Чего ожидать, а чего нет

### Реалистичные ожидания

**Что даст курс:**
- Понимание, как выбирать и использовать ИИ-инструменты
- Навык делегировать агентам рутинную разработку (CRUD, тесты, рефакторинг, документация)
- Умение координировать несколько агентов для решения сложных задач
- Полный рабочий сетап с CLI-агентом и MCP-серверами
- Освобождение времени от рутинного кодинга. Больше времени для ревью, архитектуры и бизнес-логики

**Что НЕ даст курс:**
- **Замену вашей работы** — финальная ответственность за код остаётся на вас
- **Автоматизацию 100% задач** — архитектура, бизнес-логика, интеграции без документации требуют человека
- **Глубокое понимание ML/нейросетей** — это курс про использование готовых инструментов, а не их создание
- **Волшебную палочку** — агенты ошибаются, галлюцинируют, требуют проверки

### Честно про ограничения

**Агенты полезны:**
- Рутинный CRUD
- Boilerplate код
- Тесты по шаблону
- Документация
- Рефакторинг по чётким правилам
- Генерация контента (глоссарии, лендинги)
- Написание кода по спецификации

**Агентам сложно даются:**
- Архитектурные решения (агент предложит — человек решает)
- Работа с legacy без тестов
- Интеграция с внешними API без документации
- UX-решения
- Бизнес-логика со неочевидными правилами

**Нельзя делегировать:**
- Финальная приёмка и code review
- Деплой в production
- Решения по безопасности и доступам
- Выбор архитектуры системы
- Работа с персональными данными клиентов
- Определение приоритетов бизнеса

**Принцип:** Агент делает — человек принимает. Автономность агента растёт с качеством спецификации и тестов.
