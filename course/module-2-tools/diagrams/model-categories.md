# Три категории моделей

Не "какая модель лучше", а "какая модель для какой роли". Умные модели планируют, средние реализуют, дешевые — массовая работа.

```mermaid
flowchart TD
    Task["Ваша задача"]

    Task --> Q1{"Требуется<br>глубокое<br>рассуждение?"}
    Q1 -->|"Да: архитектура,<br>безопасность,<br>ревью"| Smart["Умные модели<br>Opus, GPT-5 Pro<br>$5-25 / 1M tokens<br>Медленно, дорого, глубоко"]

    Q1 -->|"Нет"| Q2{"Стандартная<br>реализация<br>фичи?"}
    Q2 -->|"Код по спеке,<br>тесты, рефакторинг"| Medium["Средние модели<br>Sonnet, GPT-5<br>$1-15 / 1M tokens<br>Баланс скорости и качества"]

    Q2 -->|"Мелкие правки"| Cheap["Дешевые модели<br>Haiku, DeepSeek<br>$0.30-5 / 1M tokens<br>Быстро, дешево, просто"]
```

**Мульти-модельная стратегия экономит 50-80%:**
- Opus планирует архитектуру (10% задач, 40% бюджета)
- Sonnet реализует фичи (60% задач, 50% бюджета)
- Haiku/DeepSeek делает boilerplate (30% задач, 10% бюджета)
