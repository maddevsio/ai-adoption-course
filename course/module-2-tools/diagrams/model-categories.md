# Три категории моделей

Не "какая модель лучше", а "какая модель для какой роли". Умные модели планируют, средние реализуют, дешевые — массовая работа.

```mermaid
flowchart TD
    Task["Ваша задача"]

    Task --> Q1{"Требуется\nглубокое\nрассуждение?"}
    Q1 -->|"Да: архитектура,\nбезопасность,\nревью"| Smart["Умные модели\nOpus, GPT-5 Pro\n$5-25 / 1M tokens\nМедленно, дорого, глубоко"]

    Q1 -->|"Нет"| Q2{"Стандартная\nреализация\nфичи?"}
    Q2 -->|"Да: код по спеке,\nтесты, рефакторинг"| Medium["Средние модели\nSonnet, GPT-5\n$1-15 / 1M tokens\nБаланс скорости и качества"]

    Q2 -->|"Массовая обработка\nили мелкие правки"| Cheap["Дешевые модели\nHaiku, DeepSeek\n$0.30-5 / 1M tokens\nБыстро, дешево, просто"]
```

**Мульти-модельная стратегия экономит 50-80%:**
- Opus планирует архитектуру (10% задач, 40% бюджета)
- Sonnet реализует фичи (60% задач, 50% бюджета)
- Haiku/DeepSeek делает boilerplate (30% задач, 10% бюджета)
