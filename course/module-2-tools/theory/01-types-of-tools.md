# Модуль 2: Ландшафт инструментов

> **Цель:** Выбирать AI-инструмент под задачу, понимать экономику моделей.

> [Диаграмма: Выбор инструмента по задаче (дерево решений)](../diagrams/tool-selection.md)

## 1. Четыре категории инструментов

### 1.1. LLM-чаты (ChatGPT, Claude.ai, Gemini, DeepSeek)

**Что это:** Веб-интерфейсы для общения с языковыми моделями. Вы пишете запрос, получаете ответ, можете уточнять в диалоге.

**Когда можно использовать:**
- **Исследование** — "Как работает OAuth 2.0?", "Какие библиотеки есть для WebSocket в Python?"
- **Быстрые вопросы** — "Что такое SDD?"
- **Brainstorming** — "Предложи варианты имплементации для real-time чата на 10K пользователей"
- **Обучение** — разбор концепций, объяснение паттернов проектирования
- **Документация** — генерация README, API описаний

**Ограничения:**
- Нет прямого доступа к файловой системе (вы копируете код вручную)
- Нет автоматизации (каждый запрос — ручная работа)
- Контекст теряется между сессиями (если не сохранять в Projects/Memory)

---

### 1.2. IDE-копилоты (GitHub Copilot, Tabnine, Codeium)

**Что это:** Расширения для вашей IDE (VS Code, JetBrains, Neovim), которые подсказывают код во время набора

**Когда использовать:**
- **Автодополнение**
- **Рутинный код** — тесты, геттеры/сеттеры, CRUD-операции

**Ограничения:**
- **Реактивность** — вы реагируете на предложения, а не ставите задачи
- **Масштаб** — генерирует фрагменты кода, а не решает целые задачи
- **Контекст** — ограничен открытыми файлами в IDE

---

### 1.3. AI-IDE (Cursor, Antigravity, Kilo Code)

**Что это:** IDE со встроенными AI-агентами. Читают файлы, пишут код, запускают команды автономно.

**Когда использовать:**
- **Агентный режим** — "Добавь аутентификацию JWT в этот FastAPI проект" (агент сам найдет нужные файлы, добавит код, создаст миграции)
- **Multi-file changes** — рефакторинг, переименование, изменение структуры
- **Написание модулей** — делегирование целых задач ("создай CRUD для модели User")

**Ограничения:**
- **Масштаб** — эффективны для задач внутри одного проекта, сложнее для multi-repo
- **Контроль** — агент может "уйти не туда", требуется надзор
- **Цена** — более дорогие модели (Opus, GPT-5) быстро съедают бюджет в агентном режиме

> [!TIP]
> Начинайте с конкретных, четко ограниченных задач. Вместо "улучши этот проект" давайте "добавь валидацию email в форму регистрации, файл src/auth/register.py". Узкая область = меньше ошибок, быстрее feedback.

---

### 1.4. CLI-агенты (Claude Code, OpenCode, Codex CLI, Kimi CLI)

**Что это:** CLI-агенты для автономной разработки. Задача в терминале → агент читает файлы, пишет код, запускает тесты, коммитит.

CLI-агент, в отличие от AI-IDE, меньше зависит от человека. Можно снять ограничения (`--dangerously-skip-permissions`) для полной автономности. Когда агент задаст вопрос или запросит ревью, проверяйте его работу

**Когда использовать:**
- **Автономная работа** — агент работает в фоне, вы делаете другие задачи

**Ограничения:**
- Масштабируется лучше, чем AI-IDE, но все еще не может работать полностью автономно. Нужно ревью от человека
