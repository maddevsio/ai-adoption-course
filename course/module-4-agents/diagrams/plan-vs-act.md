# Режимы работы: Plan vs Act

Два режима — это не просто фазы одной задачи, а разные уровни полномочий агента. В Plan-режиме агент только читает и анализирует. В Act-режиме — читает, пишет и исполняет.

```mermaid
flowchart TB
    subgraph plan["Plan Mode (только чтение)"]
        P1["Чтение файлов проекта"]
        P2["Анализ архитектуры"]
        P3["Поиск зависимостей"]
        P4["Формирование плана"]
        P1 --> P2 --> P3 --> P4
    end

    subgraph act["Act Mode (чтение + запись)"]
        A1["Создание / редактирование файлов"]
        A2["Запуск команд (тесты, билд)"]
        A3["Итерация при ошибках"]
        A4["Коммит результата"]
        A1 --> A2 --> A3 --> A4
    end

    Start["Новая задача"] --> plan
    plan -->|"Вы одобряете\nплан"| act
    act --> Done["Результат\nготов к ревью"]
```

**Зачем разделять:**
- **Plan** использует дорогую умную модель (Opus) — рассуждает, анализирует контекст
- **Act** может использовать дешевую модель (Sonnet, Haiku) — следует плану и пишет код
- Между режимами — ваша точка контроля: одобрить план или скорректировать
