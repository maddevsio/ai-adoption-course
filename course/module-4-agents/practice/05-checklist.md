# Чеклист готовности

## Что вы прошли

1. **Создали AGENTS.md** — конституцию проекта для агента
2. **Описали проблему**, получили план от агента-планировщика
3. **Агент реализовал план**, вы провели code review по чеклисту
4. **Дали обратную связь**, агент исправил проблемы

## Проверьте себя

- [ ] AGENTS.md создан и содержит все секции (Project, Stack, Structure, Conventions, Rules, Forbidden)
- [ ] Получен план от агента, проведено ревью плана
- [ ] Агент реализовал план, результат проверен по чеклисту из 6 пунктов
- [ ] Найденные проблемы исправлены через итерацию
- [ ] Все тесты проходят, линтер чистый
- [ ] Изменения закоммичены

## Ключевые паттерны

**Flow:** проблема → планировщик → план → реализация → ревью → итерация

**AGENTS.md** эволюционирует с проектом: каждая ошибка агента → новое правило

**Правило 3 итераций:** если проблема не решена за 3 попытки — смените подход

**Качество встраивается в промпт:** "Реализуй endpoint" → "Реализуй endpoint, покрой тестами, убедись что pytest проходит, проверь линтер"

## Следующие шаги

- Примените AGENTS.md к вашему реальному проекту (не только песочнице)
- **Модуль 5:** Spec-Driven Development — спецификации фич, трёхфазный подход spec → implementation → trace
- **Модуль 6:** MCP — интеграция агентов с внешними системами (Git, Jira, Figma)
- **Модуль 7:** Оркестрация — координация нескольких агентов

**Общее время модуля:** 45 минут
