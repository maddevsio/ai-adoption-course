# Пример Progress Table

Progress Table — таблица в trace-файле, которую агент обновляет после каждой итерации. Позволяет разработчику видеть прогресс и определить, на каком шаге агент застрял.

**Пример из реальной сессии:**

| # | Задача | Статус | Итерации | Результат |
|---|--------|--------|----------|-----------|
| 1 | Создать модели данных | done | 2 | `models/user.go` создан, тесты зеленые |
| 2 | Написать API endpoint | done | 3 | `handlers/user.go`, линтер ок |
| 3 | Добавить валидацию | in progress | 1 | Работа идет... |
| 4 | Интеграционные тесты | pending | 0 | -- |
| 5 | Документация API | pending | 0 | -- |

**Как читать:**
- **done** — задача завершена, все проверки пройдены
- **in progress** — агент работает над задачей прямо сейчас
- **pending** — задача ожидает своей очереди
- **Итерации** — сколько раз агент прошел цикл Do -> Check -> Fix

**Зачем нужна таблица:**
- Разработчик видит прогресс без чтения всего лога
- Если агент застрял (5+ итераций на одной задаче) — сигнал для вмешательства
- При crash/перезапуске агент продолжает с последней in progress задачи
