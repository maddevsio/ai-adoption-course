# Подготовка: Изучить источники и установить skills (10 мин)

## Чек-лист готовности

- [ ] Вы изучили теорию Модуля 5
- [ ] Claude Code установлен и работает
- [ ] Песочница task-manager или ваш проект готовы

## Источники по SDD

1. **[Spec-driven development](https://www.thoughtworks.com/en-us/insights/blog/agile-engineering-practices/spec-driven-development-unpacking-2025-new-engineering-practices)** (Thoughtworks)
   - Определение SDD, отличие от традиционной разработки, связь с TDD и BDD

2. **[Understanding SDD: Kiro, spec-kit, Tessl](https://martinfowler.com/articles/exploring-gen-ai/sdd-3-tools.html)** (Martin Fowler)
   - Сравнение трёх SDD-инструментов, три уровня строгости спецификации

## Примеры конституций

1. **[Next.js AGENTS.md](https://github.com/vercel/next.js/blob/canary/AGENTS.md)** — эталон для крупного монорепозитория
2. **[Anthropic claude-code-action CLAUDE.md](https://github.com/anthropics/claude-code-action/blob/main/CLAUDE.md)** — конституция от создателей Claude
3. **[GitHub Spec Kit AGENTS.md](https://github.com/github/spec-kit/blob/main/AGENTS.md)** — иерархия constitution → specs → plans → tasks
4. **[awesome-claude-md](https://github.com/josix/awesome-claude-md)** — коллекция AGENTS.md и CLAUDE.md из десятков проектов

## Skills для установки
https://github.com/obra/superpowers

## Задание

1. Прочитайте источники по SDD
2. Изучите минимум 2 примера конституций из списка
3. Установите skills superpowers
4. Протестируйте каждый установленный skill на простой задаче
5. Зафиксируйте заметки:
   - Какие идеи из источников показались наиболее ценными?
   - Какие паттерны повторяются в разных конституциях?
   - Какой skill показался наиболее полезным для вашего workflow?
