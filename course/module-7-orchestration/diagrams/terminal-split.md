# Параллельная работа: Terminal Split

Два агента работают одновременно в разных worktrees, каждый — над своей задачей. Результаты сливаются через PR.

```
+------------------------------+------------------------------+
| Terminal A                   | Terminal B                   |
| worktree: ../project-auth    | worktree: ../project-api     |
| branch: feat/auth            | branch: feat/api             |
+------------------------------+------------------------------+
| $ claude                     | $ claude                     |
| > Реализуй аутентификацию    | > Реализуй REST API для      |
|   по JWT согласно            |   управления пользователями  |
|   spec/auth.md               |   согласно spec/users-api.md |
|                              |                              |
| [Агент работает...]          | [Агент работает...]          |
| - Читает спеку               | - Читает спеку               |
| - Создает middleware          | - Создает handlers           |
| - Пишет тесты                | - Пишет тесты                |
| - Запускает проверки         | - Запускает проверки         |
| - Создает PR                 | - Создает PR                 |
+------------------------------+------------------------------+
```

**Как запустить:**
1. Создайте worktree: `git worktree add ../project-auth -b feat/auth`
2. Откройте второй терминал (или split в tmux)
3. В каждом терминале — `cd` в свой worktree и запустите агента
4. Агенты работают параллельно, не мешая друг другу
5. После завершения — ревью PR и merge
