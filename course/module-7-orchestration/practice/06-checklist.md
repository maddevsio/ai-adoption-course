# Чеклист готовности

## Что вы прошли

1. **Git worktree** — изоляция агентов для параллельной работы
2. **Генератор + Ревьюер** — двухэтапная проверка для повышения качества
3. **Параллельная работа** — два агента на независимых задачах + слияние результатов
4. **Maestro** — установка оркестратора для автоматического исполнения playbook-ов
5. **Playbook** — формализованный план для оркестрации агентов

## Проверьте себя

- [ ] Создали worktree, запустили агента, убедились в изоляции
- [ ] Провели цикл генератор → ревьюер → исправление
- [ ] Запустили два агента параллельно, смержили результаты
- [ ] Установили Maestro, подключили агента
- [ ] Написали playbook для задачи из своего проекта
- [ ] Все тесты проходят после merge

## Ключевые паттерны

**Одна ветка = один план = один worktree** — изоляция предотвращает 90% конфликтов

**Ralph Loop (Do → Check → Fix)** — агент работает до зелёных тестов, а не до "я написал код"

**Разные модели для разных ролей** — Opus для архитектуры, Sonnet для планирования, Haiku для кодинга

**Playbook = воспроизводимость** — формализованный план можно повторить, передать, автоматизировать через Auto Run

## Следующие шаги

1. Возьмите фичу из своего проекта и реализуйте с 2-3 агентами параллельно
2. Экспериментируйте с open-source оркестраторами
    - https://github.com/RunMaestro/Maestro
    - https://github.com/mikeyobrien/ralph-orchestrator
    - и др.
3. Попробуйте разные модели на разных типах задач — измеряйте время и стоимость
