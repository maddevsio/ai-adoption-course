# Что НЕЛЬЗЯ отправлять в контекст ИИ

Данные, которые ни при каких обстоятельствах не должны попадать в промпты или файлы, доступные агенту. Правило: если утечка = компрометация — данные не для ИИ.

```mermaid
flowchart TD
    subgraph forbidden["ЗАПРЕЩЕНО передавать агенту"]
        F1["API-ключи, токены,\nпароли, .env файлы"]
        F2["Приватные данные (PII)\nemail, телефоны,\nмедицинские записи"]
        F3["NDA-документы\nстратегия, финансы,\nнераскрытые продукты"]
        F4["Session tokens,\nJWT, cookies\n(дают доступ без пароля)"]
        F5["Закрытые алгоритмы\nесли политика компании\nзапрещает передачу"]
    end

    subgraph safe["БЕЗОПАСНО"]
        S1["Код без секретов\n(проверить grep -r\n'password|secret|key')"]
        S2["Публичные API доки\nОткрытые спеки"]
        S3["Конфигурации\nбез credentials\n(.env.example)"]
    end

    Check{"Может ли утечка\nпривести к\nкомпрометации,\nштрафу, иску?"}
    Check -->|"Да"| forbidden
    Check -->|"Нет"| safe
```

**Штрафы за утечку:**
- GDPR: до 20 млн евро или 4% глобального оборота
- HIPAA: до $1.5 млн в год
- Утечка API-ключей: несанкционированный доступ к системам
