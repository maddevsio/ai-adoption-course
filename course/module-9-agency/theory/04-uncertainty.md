# Модуль 9: Агентность разработчика

## 4. Неопределённость

LLM — недетерминированный инструмент. Один и тот же промпт может дать разные результаты. Это изначальная характеристика LLM. Кто не может с этим работать — не сможет полноценно использовать агентов.

### Аналогия с людьми

Работа с людьми тоже недетерминирована. Ты даёшь задачу джуниору — результат непредсказуем. Но ты проверяешь и итерируешь. С агентом то же самое. Разница только в скорости итераций.

### Неполное ТЗ — нормальная ситуация

Идеального ТЗ не существует. В реальных проектах всегда чего-то не хватает: нет дизайна, нет спеки API, нет описания edge cases. Инженер начинает с того, что есть, и уточняет по ходу.

### Итеративность

Первая попытка — черновик, не финал. Это справедливо для:

- Своего кода
- Кода агента
- ТЗ
- Архитектуры

Ожидать идеальный результат с первой попытки — ловушка. Даже опытный инженер не пишет финальный код с первого раза. Он пишет, запускает, правит. Агент работает по той же схеме, только быстрее.

### Практический навык

Умение начать с неполной информацией — это навык, который можно тренировать:

1. Получи задачу
2. Сформулируй первое приближение
3. Получи результат (от агента или от себя)
4. Оцени — что не так?
5. Скорректируй курс

Этот цикл работает на любом уровне: от одной функции до целой фичи. Чем быстрее ты проходишь цикл, тем быстрее добираешься до правильного решения.
