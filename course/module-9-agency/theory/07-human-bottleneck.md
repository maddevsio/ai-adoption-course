# Модуль 9: Агентность разработчика

## 7. Человек — боттлнек

Парное программирование с агентом не масштабируется. Ты по-прежнему в каждой задаче, просто быстрее. 2x, может 3x — но не 10x. Один человек + один агент = линейный рост. Для экспоненциального роста потенциала нужен переход от "работаю с агентом" к "управляю флитом агентов".

#### Где настоящий боттлнек

Боттлнек — не скорость набора кода. Боттлнек — внимание человека: формулировка задачи, проверка результата, принятие решений. Это последовательные операции, они не параллелятся.

Пока ты в каждом цикле агента — ты ограничиваешь систему собой.

#### Как убрать себя из critical path

Вынести контекст из головы в артефакты: спецификации, тесты, конституции, AGENTS.md. Агент должен работать без тебя в цикле, а не ждать твоего фидбэка на каждый шаг.

Менеджер, который делает всё сам — боттлнек команды. Тимлид, который пишет доки, процессы, чеклисты — масштабирует команду. С агентами то же самое: инвестиция в инфраструктуру контроля окупается при масштабе.

#### Инфраструктура контроля

- **Тесты** — агент сам проверяет свой результат
- **Линтеры и CI** — автоматический фидбэк без участия человека
- **Спецификации** — задача без устных пояснений
- **AGENTS.md** — правила без повторных объяснений
- **Hooks** — steering без ручного вмешательства

#### Ключевой сдвиг

От "я контролирую каждый шаг" к "я построил систему, которая контролирует за меня". Доверие через инфраструктуру, а не через надзор.
