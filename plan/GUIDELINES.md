## Язык

- Материалы курса: **русский**
- Примеры кода, команды CLI, имена файлов: **английский**
- Цитаты из Enji Fleet: сохранять оригинальный язык (могут быть и русские, и английские)
- Не переводить технические термины, у которых нет устоявшегося русского аналога: CLI, MCP, hot-swap, worktree, RAG, SDD

## Стиль текста

- Прямой, конкретный, без воды
- Каждое утверждение подкреплено примером или ссылкой
- Запрещённые слова: "революция", "трансформация", "будущее уже здесь", "game-changer", "парадигмальный сдвиг"
- Не использовать emoji в тексте
- Первое лицо множественного числа ("мы", "наши") для обращения к аудитории
- "Ты" для прямых инструкций ученику
- Без docstrings и комментариев в примерах кода (код самодокументируемый)

## Формат файлов

- Markdown (.md)
- Заголовки: `#` для названия, `##` для секций, `###` для подсекций
- Блоки кода: с указанием языка (```python, ```bash, ```yaml, ```go)
- Таблицы: для сравнений и списков с несколькими параметрами
- Длина строки: без жёсткого лимита, но абзацы не длиннее 5-6 строк

## Работа с Enji Fleet

Enji Fleet -- сквозной пример курса. Репозиторий: `/home/kb/Documents/workspace/mad_devs/enji-fleet`

Правила:
1. Читай `sources/enji-fleet-analysis.md` для понимания, какие практики где использовать
2. Ходи в репозиторий Fleet за **конкретным кодом и текстом**
3. Цитируй файлы с путями: "Из `AGENTS.md` проекта Fleet:"
4. Не пересказывай абстрактно -- показывай реальный код/конфиг
5. Маппинг практика→модуль описан в enji-fleet-analysis.md, следуй ему

## Критерии приёмки

Задача считается выполненной когда:
1. Файл создан по указанному пути
2. Все пункты из секции "Что написать" покрыты
3. Все пункты из секции "Критерии приёмки" выполнены
4. Объём текста в пределах ±15% от указанного
5. Примеры из Enji Fleet использованы (если указано "обязательно")
6. Нет разрыва с материалами предыдущих модулей (читай COURSE_STRUCTURE.md)

## Зависимости

Не начинай задачу, если её зависимости не выполнены. Проверяй наличие файлов:
- Задача 04 зависит от задачи 02 → проверь `research/case-studies.md`
- Задача 05 зависит от 01, 03 → проверь `research/tools-comparison.md`, `research/stack-decision.md`
- Практические задачи (16-19) зависят от `course/test-project/`
