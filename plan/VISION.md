# Vision: Курс по внедрению ИИ в работу разработчиков

## Анализ исходных материалов

Изучены три документа:

1. **Портрет ИИ зрелости** — модель из 6 уровней (чат-кодинг → оркестратор), матрица компетенций, психопрофиль, вопросы для найма. HR-инструмент, из которого берём целевые уровни и критерии.

2. **Контент-план по курсу ИИ** — черновой план на 4 модуля (~90 мин). Хороший скелет, темы правильные, но хронометраж нереалистичный. При расширенном бюджете (8ч) можно покрыть всё, что заложено в контент-плане.

3. **Уровни AI usage maturity** — старая модель на 5 уровней, более общая (все роли, не только разработчики). Хорошее описание ответственности на каждом уровне, нуждается в актуализации.

## Корректировка с реальностью рынка (февраль 2026)

Что изменилось с момента написания исходных документов:

- **MCP стал стандартом** — 97M+ загрузок SDK, 5800+ серверов. Базовая инфраструктура, а не экзотика.
- **CLI-агенты стали мейнстримом** — Claude Code, Aider (39K stars, 4.1M установок), OpenCode (95K stars).
- **Мульти-модельная стратегия** — норма. Claude Opus 4.6 для сложной логики, GPT-5.2 для универсальных задач, Gemini 3 Pro для большого контекста, локальные модели для приватности.
- **Spec-driven development** — признанная практика. Anthropic написали компилятор C на Rust этим методом.
- **Cursor** — зрелые продукты с агентными режимами.
- **Оркестрация** — не теория, а практика. Claude Code с субагентами, Cursor с параллельными composer-сессиями, git worktrees для изоляции.
- **Реалистичные ожидания**: ИИ помогает в ~60% работы, полностью автоматизирует 0-20%. Остальное — человек.

Что это значит для курса:

- Инструменты доступнее, чем год назад. Порог входа снизился.
- Агентный режим — базовый навык для 2026.
- MCP — обязательный компонент рабочего сетапа, не опция.
- Оркестрация достижима для разработчика среднего уровня при правильном обучении.
- Spec-driven development — практичная методология, которая хорошо ложится на агентную работу.

## Scope курса

### Целевая аудитория

Разработчик ПО в компании Mad Devs. На входе:
- Умеет программировать (любой стек)
- Слышал про ChatGPT, возможно использует для вопросов
- Уровень 1-2 по модели зрелости (чат-кодинг / копилот в IDE)

### Целевой результат

На выходе — уровень 5 (менеджер параллельных агентов / оркестратор):
- Полный рабочий сетап: AI-IDE + CLI-агент + MCP-серверы + настроенные правила
- Умеет ставить задачи одному агенту и проверять результат
- Умеет настраивать MCP для расширения возможностей агента
- Владеет spec-driven development: пишет спецификации, по которым агент работает
- Умеет запускать параллельные агентские сессии и координировать их
- Понимает паттерны оркестрации (Ralph Loop, генератор-ревьюер, иерархии)
- Знает риски и границы ответственности на каждом уровне автономности

### Путь ученика через уровни зрелости

```
Уровень 1-2          Уровень 3              Уровень 4              Уровень 5
Чат / Копилот   →    Один агент       →     Агент + MCP + SDD  →   Оркестрация
(спрашиваю)          (делегирую)            (расширяю)              (координирую)
Модули 1-3           Модуль 4               Модули 5-6             Модуль 7
```

### Что в scope

- Уровни 1 → 5: от чата до оркестрации параллельных агентов
- Промптинг: от базового к системным промптам и спецификациям
- Инструменты: CLI-агент (Claude Code — основной) + AI-IDE (Cursor/Copilot — опционально)
- Управление контекстом: AGENTS.md, .cursorrules, project constitution, feature plans
- Spec-driven development: написание спецификаций для агентов
- MCP: установка, настройка, выбор серверов, практическое применение
- Оркестрация: Ralph Loop, параллельные сессии, паттерн генератор-ревьюер
- Инфраструктура параллельной разработки: Claude Code в облаке, headless-режим, auth-токены, Claude Agent SDK, OpenClaw
- Ответственное использование: риски, безопасность, ответственность

### Что за scope

- Уровень 6 (проектирование автономных систем, кастомные фреймворки)
- Написание собственных MCP-серверов
- Локальные модели (Ollama) — упоминание, не установка
- Построение production-пайплайнов CI/CD с агентами (обзорно — да, полный setup — нет)
- LangChain / LangGraph / AutoGPT фреймворки

## Принципы курса

1. **Практичность** — каждый теоретический блок ведёт к конкретному действию на машине.
2. **Прогрессия** — от простого к сложному, каждый модуль строится на предыдущем.
3. **Понятность** — термины объясняются через действия, не через определения.
4. **Честность** — ИИ не магия. Говорим про ограничения, ошибки, стоимость.
5. **Актуальность** — инструменты и примеры из 2026.
6. **Полный сетап** — к концу курса на машине ученика всё установлено и настроено.
