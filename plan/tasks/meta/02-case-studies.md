# Задача 02: Подготовка кейсов с метриками

## Цель

Найти 3 реальных кейса использования ИИ в разработке с конкретными метриками. Кейсы используются в Модуле 1 для мотивации.

## Что сделать

1. Найти 3 кейса, подходящих под критерии:
   - Реальная компания (не стартап из 2 человек)
   - Конкретные метрики (время, стоимость, качество — хотя бы одно)
   - Релевантно разработке ПО (не маркетинг, не дизайн)
   - Свежие (2025-2026)

2. Для каждого кейса собрать:
   - Компания и контекст
   - Что делали (какие инструменты, какой процесс)
   - Метрики до/после
   - Источник (ссылка)

3. Один из кейсов желательно из опыта Mad Devs или похожей аутсорс-компании.

## Формат результата

MD-файл `research/case-studies.md` с 3 кейсами, каждый на 3-5 абзацев.

## Критерии приёмки

- 3 кейса с проверяемыми метриками
- Источники указаны и доступны
- Минимум 1 кейс про агентную разработку (не просто копилот)
- Кейсы разнообразны (разные масштабы, разные инструменты)

## Зависимости

Нет.

### Внутренний кейс: Enji Fleet (Mad Devs)

**Обязательный кейс.** Enji Fleet -- реальный проект Mad Devs, платформа оркестрации AI-агентов, которая сама разрабатывается агентами. Подробный анализ: `sources/enji-fleet-analysis.md`. Репозиторий: `/home/kb/Documents/workspace/mad_devs/enji-fleet`.

Что показывает:
- Проект на Go/React с 7 микросервисами, написанными агентами через 16+ планов
- 4 AI CLI в одном Docker-образе (Claude, Codex, Gemini, OpenCode)
- Constitution как "память" между сессиями агентов
- Trace-файлы как эволюционный механизм накопления знаний
- Hot-swap агентов при rate limit или зацикливании
- Обязательный git worktree для изоляции параллельных агентов
- MCP-сервер для коммуникации агент↔платформа

Метрики для кейса: количество планов (16+), количество сервисов (7), количество traces (25+), поддерживаемых AI CLI (4), embedding-провайдеров (3).
