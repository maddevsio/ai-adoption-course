# Задача 03: Фиксация стека инструментов для курса

## Цель

Выбрать и зафиксировать конкретные инструменты, версии и конфигурации, которые будут использоваться в практической части курса.

## Что сделать

1. На основе исследования (задача 01) выбрать:
   - **Основной инструмент**: Claude Code CLI (терминальный агент)
   - **Дополнительная IDE** (опционально): Cursor, VS Code + Copilot
   - **Модели**: мульти-модельная стратегия (умные → средние → дешёвые), зафиксировать конкретные модели для каждой роли

2. Для каждого выбранного инструмента зафиксировать:
   - Версию (или "latest stable" если инструмент быстро обновляется)
   - Минимальные системные требования
   - Стоимость (какую подписку нужно оформить)
   - Инструкцию по получению API-ключа / подписки

3. **MCP-серверы** для курса: 4 обязательных сервера из исследования (задача 01):
   - git, jira, IDE (PyCharm debugger), Figma
   - Зафиксировать способ установки
   - Формат конфигурации для Claude Code (и Cursor если используется)
   - Порядок установки в курсе

4. **Инфраструктура параллельной разработки**: на основе Части B+ задачи 01:
   - Headless-режим: какие флаги документируем
   - Auth-токены: API key vs setup-token, рекомендация для курса
   - Agent SDK: показываем или упоминаем?
   - Agent Teams: показываем как experimental или пропускаем?
   - OpenClaw: глубина покрытия (обзор vs практика)

5. Принять решение по спорным вопросам:
   - Платные подписки: компания оплачивает или сотрудник?
   - macOS / Linux / Windows: какие ОС поддерживаем?
   - Какие MCP-серверы обязательны, какие опциональны?
   - Бесплатные альтернативы: для кого (стажёры, личные проекты)?

## Формат результата

MD-файл `research/stack-decision.md` с зафиксированными решениями и обоснованиями.

## Критерии приёмки

- Каждый выбор обоснован (почему X, а не Y)
- Указана стоимость на сотрудника в месяц
- Указаны поддерживаемые ОС
- Нет открытых вопросов — все решения приняты
