# Задача 15: Упражнения по промптингу

## Цель

3 практических упражнения для Модуля 3.

## Что сделать

### Упражнение 1: Улучши промпт (10 мин)

3 "плохих" промпта → улучшить → запустить оба → сравнить.

Примеры плохих промптов (использовать как основу):
1. **Функция**: "Напиши функцию для валидации" — нет контекста: что валидировать, какой язык, какие правила, куда положить
2. **Рефакторинг**: "Отрефактори этот код чтобы было лучше" — нет критериев "лучше", нет scope, нет ограничений
3. **Тесты**: "Напиши тесты" — нет указания что тестировать, какой coverage, какие edge cases, какой фреймворк

Эталонные "хорошие" варианты для каждого.
Чеклист: что делает промпт "хорошим" (контекст, задача, формат, ограничения, критерии приёмки)

### Упражнение 2: Промпты для разных задач (15 мин)

Написать промпты для 3 типов задач на своём проекте:
- Написать новую функцию/модуль
- Написать тесты для существующего кода
- Отрефакторить существующий код

Шаблон промпта для каждого типа. Чеклист самопроверки.

### Упражнение 3: Игра Gandalf (15 мин)

Ссылка: https://gandalf.lakera.ai/intro

Задача: пройти уровни, пытаясь заставить ИИ выдать секретный пароль.
- Каждый уровень — новая защита, нужны всё более хитрые промпты
- Цель: почувствовать границы промптинга, научиться обходить ограничения
- После прохождения: рефлексия — какие техники сработали? Что это говорит о работе с ИИ?

## Критерии приёмки

- 3 плохих промпта из реальной разработки

- Шаблоны для разных типов задач
- Инструкция для Gandalf + рефлексивные вопросы после
- Суммарно 40 мин
