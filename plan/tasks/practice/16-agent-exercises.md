# Задача 16: Упражнения по работе с агентом

## Цель

4 практических шага для Модуля 4. Путь: настройка AGENTS.md → описать проблему → агент-планировщик декомпозирует → реализация → ревью → итерация.

## Что сделать

### Шаг 1: Создать AGENTS.md (10 мин)

- Шаблон AGENTS.md с комментариями (не CLAUDE.md — используем AGENTS.md как основное имя)
- Ученик создаёт для своего проекта (или песочницы из задачи 14)
- Эталонный AGENTS.md для сверки
- Минимальные секции: Project, Stack, Conventions, Rules
- Пояснить: AGENTS.md = CLAUDE.md = .cursorrules — одна идея, разные имена

### Шаг 2: Описать проблему и получить план (15 мин)

Новый flow (не "вот тебе готовый промпт"):
1. Ученик описывает проблему своими словами: "Нужен эндпоинт для обновления задачи" (или страница, или отчёт)
2. Ставит задачу агенту-планировщику: "Опиши задачу, разбей на мелкие подзадачи, для каждой укажи критерии приёмки"
3. Агент выдаёт план с подзадачами
4. Ученик ревьюит план
5. Дополняет: "Добавь к каждой подзадаче: какие тесты написать, какие линтер-правила проверить"
6. Агент реализует

Подготовить:
- Пример описания проблемы
- Промпт для агента-планировщика
- Пример ожидаемого плана от агента
- Чеклист: что должно быть в плане (тесты, линтер, критерии приёмки)

### Шаг 3: Ревью результата (10 мин)

Чеклист ревью (6-8 пунктов):
- Код работает?
- Тесты проходят?
- Линтер чистый?
- Стиль соответствует AGENTS.md?
- Нет лишних файлов/зависимостей?
- Нет security-проблем?

"Подводные камни": что агент типично делает не так.

### Шаг 4: Итерация (10 мин)

- Ученик берёт найденные проблемы из ревью
- Ставит агенту задачу исправить, ссылаясь на конкретные проблемы
- Повторное ревью

## Формат результата

MD-файл `course/module-4-agents/practice.md`

## Критерии приёмки

- AGENTS.md (не CLAUDE.md) как основной артефакт
- Flow: проблема → планировщик → план → реализация → ревью (не "вот готовый промпт")
- Агент сразу фреймится на качество (тесты, линтер, критерии)
- Чеклист ревью 6-8 пунктов
- Суммарно 45 мин
