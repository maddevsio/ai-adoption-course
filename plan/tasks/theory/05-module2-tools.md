# Задача 05: Модуль 2 — Ландшафт инструментов (теория)

## Цель

Написать текст теоретической части Модуля 2. Дать ученику карту инструментов, включая бесплатные, и стратегию выбора моделей.

## Что написать

Текст на ~800 слов + таблица сравнения.

1. **4 категории инструментов** (по 1-2 абзаца на каждую):
   - LLM-чаты (ChatGPT, Claude.ai, Gemini) — когда: исследование, вопросы, brainstorming
   - IDE-копилоты (GitHub Copilot, Tabnine) — когда: автодополнение, быстрые подсказки
   - AI-IDE (Cursor, Windsurf, Google Antigravity) — когда: агентный режим, работа с кодовой базой
   - CLI-агенты (Claude Code, Codex CLI, Aider, OpenCode) — когда: сложные задачи, автономная работа

2. **Бесплатные и дешёвые варианты** (1-2 абзаца):
   - **OpenCode** (103K stars) — open-source CLI-агент, поддерживает все провайдеры (Claude, GPT, Gemini, Ollama), бесплатен, платишь только за API ключи. Альтернатива Claude Code без привязки к Anthropic.
   - **Claude free tier** — 30-100 сообщений/день на Sonnet/Haiku без подписки. Guest Pass (от Max-подписчиков) даёт 7 дней Pro-доступа. Хватает для обучения, не хватает для ежедневной работы.
   - **Ollama** — запуск моделей локально (Llama 3, DeepSeek, Qwen). Бесплатно, данные не уходят. Нужен GPU и 8+ GB RAM. Качество ниже облачных моделей.
   - **Kilo Code** (15K stars) — open-source расширение VS Code, fork Cline/Roo Code. Агентные режимы (Architect, Code, Debug, Orchestrator), 500+ моделей включая Ollama. $8M инвестиций, 1.5M+ пользователей.
   - **Google Antigravity** (https://antigravity.google/) — IDE с multi-agent orchestration, async tasks, бесплатно. Gemini 3, Claude, GPT-OSS.
   - **Codex CLI** (https://developers.openai.com/codex/cli/) — CLI-агент от OpenAI. GPT-5-Codex, multimodal (screenshots), enterprise-фокус.
   - **DeepSeek** — cost-sensitive API/Chat, open-source, self-hosting. DeepSeek V3/R1. Хорош для дешёвых exploratory задач и черновой работы.
   - Для каждого: что можно, чего нельзя, ограничения
   - **Комбинирование инструментов** — норма: DeepSeek для дешёвых exploratory задач, Claude Code для сложной автоматизации, ChatGPT для объяснений и документации

3. **Модели и мульти-модельная стратегия** (2-3 абзаца):
   - Не "какая модель лучше", а "какая модель для какой роли":
     - Умные (Opus, GPT-5.2) → планирование, архитектурные решения, сложная логика
     - Средние (Sonnet, GPT-4o) → планирование в цикле, ревью, итерации
     - Дешёвые/быстрые (Haiku, mini, open-source DeepSeek/Qwen/Llama) → реализация мелких задач массово
   - Стратегия: разбить задачу на минимальные части → раздать дешёвым моделям → ревью умной моделью
   - Стоимость: порядок цен на токены для каждой категории

4. **Как выбирать: задача → инструмент** (1-2 абзаца):
   - Простой вопрос → чат
   - Автодополнение → копилот
   - Написать модуль → CLI-агент или AI-IDE
   - Массовая генерация → дешёвые модели + ревью

5. **Таблица сравнения**: инструмент / категория / цена / для чего подходит / модели

## Тон и стиль

- Практичный, без рекламы
- Если инструмент имеет недостатки — сказать
- Бесплатные варианты — не "дешёвая замена", а полноценные инструменты с ограничениями
