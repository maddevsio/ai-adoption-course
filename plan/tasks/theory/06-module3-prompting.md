# Задача 06: Модуль 3 — Промптинг (теория)

## Цель

Написать текст теоретической части Модуля 3. Научить формулировать задачи для ИИ. Фокус на продвинутых, нетривиальных техниках.

## Что написать

Текст на ~800 слов.

1. **Ключевой сдвиг** (1-2 абзаца):
   - Разница между "вопрос" и "задание"
   - "Как сделать авторизацию?" → "Реализуй JWT-авторизацию для FastAPI, используя модель User из models.py, с эндпоинтами /login и /refresh"
   - Это главная мысль модуля: переход от уровня 1-2 к уровню 3

2. **Структура промпта** (2-3 абзаца):
   - Четырёхблочная структура для coding agents: **роль** (кем является агент и какие у него компетенции) → **контекст** (проект, стек, ограничения) → **задача** (что конкретно сделать) → **критерии приёмки** (как понять что задача выполнена правильно)
   - Альтернативный фреймворк — PPFO: Purpose, Planning, Format, Output
   - Чем точнее критерии приёмки, тем меньше итераций. Для агентных инструментов промпт определяет не просто ответ, а целую цепочку действий
   - Каждый элемент с примером

3. **Продвинутые техники** (по 1-2 абзаца каждая):
   - Ролевая постановка: "Ты — senior backend developer, специализирующийся на..."
   - Few-shot: дать примеры ожидаемого результата перед запросом
   - Другие нетривиальные техники (исследовать и включить):
     - Prompt chaining: разбивка сложного запроса на цепочку
     - Self-consistency: попросить проверить свой ответ
     - Constraint prompting: жёсткие ограничения формата
   - Для каждой: когда использовать, пример, почему это нетривиально

4. **3 антипаттерна** (по 1 абзацу каждый):
   - Нечёткие формулировки: "сделай красиво", "оптимизируй"
   - God-prompt: вся задача в одном огромном промпте
   - Отсутствие контекста: просьба без указания стека, проекта, ограничений

5. **Промптинг для разных задач** (2 абзаца):
   - Разработка, рефакторинг, тесты, документация, планирование — у каждого своя специфика
   - Короткий пример промпта для каждого типа задачи

## Тон и стиль

- Практичный, с реальными примерами кода
- Примеры — на разных языках (Python, Go, TypeScript)
- Фокус на нетривиальном: базовые техники (chain-of-thought, пошаговые инструкции) модели уже делают сами. Учим то, что модель сама не предложит.

## Критерии приёмки

- Ключевой сдвиг "вопрос → задание" явно сформулирован
- Структура промпта с примером каждого элемента
- Минимум 3 продвинутых техники (не базовые)
- 3 антипаттерна с примерами
- Примеры из реальной разработки
