### Кастомный MCP-сервер для оркестрации
Fleet SDK реализует собственный MCP-сервер (`backend/cmd/fleet-sdk/internal/cli/mcp_server.go`) на stdin/stdout, который предоставляет агенту инструменты для коммуникации с платформой:

| Tool | Назначение |
|------|------------|
| `report_progress` | Агент докладывает о прогрессе |
| `report_error` | Агент сообщает об ошибке |
| `store_memory` | Сохранить знание в Memory Service (ChromaDB) |
| `fetch_context` | Получить контекст через векторный поиск |
| `save_artifact` | Сохранить артефакт в Store Service (MinIO) |

Это пример того, как MCP используется не только для подключения готовых серверов (git, jira), но и для построения собственных интеграций. Агент не знает, что работает внутри оркестратора -- он просто видит дополнительные tools через MCP.
