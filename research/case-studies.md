# Реальные кейсы использования ИИ в разработке ПО

Три реальных кейса использования искусственного интеллекта в разработке программного обеспечения с конкретными метриками и результатами. Все кейсы относятся к 2025-2026 годам и демонстрируют разные подходы к внедрению ИИ: от агентной разработки до платформенных решений.

---

## Кейс 1: Enji Fleet (Mad Devs) — платформа оркестрации AI-агентов, разрабатываемая агентами

**Компания и контекст**

Mad Devs — аутсорс-компания по разработке ПО, создающая Enji Fleet — платформу для оркестрации AI-агентов в процессе разработки. Уникальность проекта в том, что сама платформа разрабатывается AI-агентами, что позволяет тестировать и валидировать практики агентной разработки в реальных условиях. Проект представляет собой систему из 7 микросервисов на Go и React с полноценной инфраструктурой: Task Service, Memory Service, Agent Service, Fleet SDK и другие компоненты.

**Технологический стек и процесс**

Платформа использует четыре различных AI CLI (Claude Code, Codex, Gemini CLI, OpenCode), упакованных в единый Docker-образ с унифицированным интерфейсом через Fleet SDK. Агенты работают в четырех режимах: dev-mode (разработка по спецификациям), doc-mode (документация), reflect-mode (аудит и консолидация знаний), push-mode (коммит и PR). Ключевой практикой является "Constitution" — живой документ объёмом ~400-500 строк, который служит "памятью" между сессиями агентов, поскольку агенты не сохраняют контекст между запусками. Каждый выполненный план оставляет trace-файл с описанием решений, проблем и рекомендаций для обновления Constitution.

Процесс разработки автономен: агент получает промпт с полным pipeline (setup → read constitution → create worktree → find task → code + tests → verify → update status → trace → PR). Обязательное использование git worktree обеспечивает изоляцию параллельных агентов. Система hot-swap автоматически переключает агентов при timeout, ошибках провайдера или провале верификации. Memory Service с векторным поиском (ChromaDB + embeddings от OpenAI/Ollama/Gemini) предоставляет RAG для агентов через MCP-инструменты.

**Метрики и результаты**

За период разработки выполнено 16+ планов, создано 7 микросервисов, накоплено 25+ trace-файлов. Платформа поддерживает 4 AI CLI провайдера с возможностью hot-swap между ними. Реализовано 3 embedding-провайдера для Memory Service. Все коммиты содержат атрибуцию "Co-Authored-By: Claude Opus 4.5" для прозрачности. Проект демонстрирует полный цикл агентной разработки: от автономного написания кода до автоматического создания PR с Telegram-постами для команды. Constitution эволюционирует через reflect-mode агента, который консолидирует знания из traces, обеспечивая накопление практик без хаотичного редактирования базового документа.

## Кейс 2: Rakuten — сокращение time-to-market на 79% с Claude Code

**Компания и контекст**

Rakuten — японская IT-корпорация, один из крупнейших e-commerce операторов в мире с годовым оборотом более $15 млрд. В 2025 году компания активно инвестировала в AI-инициативы, включая запуск собственной платформы Rakuten AI и разработку крупнейшей в Японии языковой модели. Инженерные команды Rakuten столкнулись с необходимостью ускорить вывод новых функций на рынок в условиях растущей конкуренции в e-commerce сегменте. Традиционный процесс разработки занимал в среднем 24 рабочих дня от идеи до релиза функции.

**Технологический стек и процесс**

Для решения проблемы Rakuten внедрила Claude Code от Anthropic в процесс разработки. Один из показательных тестов: инженеры поручили Claude Code реализовать метод извлечения activation vectors в vLLM — кодовой базе объёмом 12.5 миллионов строк кода. Claude Code автономно завершил задачу за 7 часов работы, достигнув 99.9% числовой точности. По словам Manoj Desai, менеджера секции AI Empowerment в Rakuten, Claude Code дал командам "суперспособности делать выполнение намного, намного быстрее".

Агентный подход позволил делегировать рутинные задачи по написанию кода, рефакторингу и тестированию AI-агенту, в то время как разработчики сосредоточились на архитектурных решениях и бизнес-логике. Claude Code интегрировался в существующие процессы через IDE (VS Code) и GitHub, обеспечивая непрерывную поддержку на всех этапах разработки.

**Метрики и результаты**

Среднее время вывода новых функций на рынок сократилось с 24 рабочих дней до 5 дней — снижение на 79%. Это позволило Rakuten значительно ускорить выпуск обновлений для своей e-commerce экосистемы и быстрее реагировать на потребности пользователей. В тестовом задании с vLLM автономная работа заняла 7 часов с точностью 99.9%, что демонстрирует возможность агентов работать с крупными кодовыми базами без постоянного надзора. Компания планирует масштабировать использование агентного AI в 2026 году, опираясь на полученный опыт и измеримые улучшения в скорости разработки.

## Кейс 3: TELUS — экономия 500K часов и ускорение shipping на 30%

**Компания и контекст**

TELUS — один из крупнейших телекоммуникационных и healthcare-провайдеров в мире, обслуживающий миллионы клиентов в Канаде и международных рынках. Компания имеет 57,000 сотрудников и масштабную инженерную инфраструктуру, где скорость и надежность критически важны. В условиях роста требований к цифровым сервисам и сокращения time-to-market TELUS искала способы повысить продуктивность инженерных команд без пропорционального роста штата.

**Технологический стек и процесс**

TELUS разработала внутреннюю AI-платформу Fuel iX, использующую Claude от Anthropic в качестве основного движка. Платформа предоставляет доступ к AI всем 57,000 сотрудникам компании, но особенно активно используется разработчиками. Инженеры получили интеграцию Claude Code непосредственно в VS Code и GitHub для рефакторинга кода в реальном времени, автоматизации тестирования и генерации документации. Команды создали более 13,000 кастомных AI-решений для специфических задач: от автоматизации CI/CD pipeline до генерации API-клиентов и миграции legacy-кода.

Платформа поддерживает различные сценарии использования: от простых code suggestions до автономного выполнения сложных задач по рефакторингу multi-module проектов. Разработчики делегируют AI-агентам рутинные задачи (написание unit-тестов, документирование API, обновление зависимостей), сохраняя контроль над архитектурными решениями. Pilot-программы показали 30% ускорение времени обработки pull request благодаря автоматизированному code review и suggestions.

**Метрики и результаты**

TELUS достигла экономии более 500,000 инженерных часов суммарно благодаря внедрению AI в процессы разработки. Скорость выпуска кода (code shipping) ускорилась на 30% по сравнению с периодом до внедрения Claude Code. Создано более 13,000 кастомных AI-решений, адаптированных под специфические задачи различных команд. Все 57,000 сотрудников получили доступ к Fuel iX платформе, что демонстрирует масштаб enterprise-внедрения AI.

Экономический эффект складывается из нескольких факторов: сокращение времени на рутинные задачи, ускорение code review процесса, снижение количества багов благодаря AI-assisted тестированию, и быстрая онбординг новых разработчиков через AI-генерированную документацию. TELUS продолжает масштабировать использование AI, планируя расширить возможности Fuel iX в 2026 году.

## Выводы

Три кейса демонстрируют разные уровни зрелости внедрения ИИ в разработку:

1. **Enji Fleet** — экспериментальная агентная разработка с полной автономией агентов и self-hosting подходом
2. **Rakuten** — внедрение агентных инструментов для ускорения конкретных процессов разработки
3. **TELUS** — масштабное enterprise-внедрение AI-платформы на уровне всей организации

Общий паттерн: переход от "AI как помощник" к "AI как автономный агент", способный выполнять сложные задачи с минимальным надзором. Метрики показывают существенные улучшения в скорости разработки (30-79% ускорение) и экономии ресурсов (500K+ часов).
